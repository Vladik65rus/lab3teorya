## 📝 Постановка задачи

Разработать синтаксический анализатор (парсер) в соответствии с индивидуальным вариантом задания, интегрировать его в приложение из лабораторной работы №1 и обеспечить наглядный вывод результатов анализа.

### Требования к разработке парсера:

1. **Разработать грамматику** для заданной синтаксической конструкции
2. **Построить схему метода анализа** на основе разработанной грамматики
3. **Выполнить программную реализацию** алгоритма работы синтаксического анализа
4. **Реализовать алгоритм нейтрализации** синтаксических ошибок методом Айронса
5. **Входные данные** — строка (текст программного кода) из области редактирования
6. **Выходные данные:**
   - При успешном анализе корректной строки — сообщение об отсутствии ошибок
   - При обнаружении ошибок — таблица с описанием каждой ошибки

### Требования к интеграции и интерфейсу:

- Встроить парсер в ранее разработанный интерфейс и связать его с кнопкой «Синтаксический анализ»
- Окно вывода результатов должно содержать таблицу ошибок со столбцами:
  - Неверный фрагмент (символ или фрагмент, вызвавший ошибку)
  - Местоположение (номер строки, позиция символа)
  - Описание ошибки
- Отображать общее количество найденных ошибок
- Реализовать навигацию по ошибкам: при щелчке на строке таблицы курсор в области редактирования устанавливается на позицию ошибочного фрагмента

## Разработка грамматики

### Терминалы (V_T):

`fun, return, Int, (, ), {, }, ;, :, ,, +, -, *, /, id, num`

### Полное определение грамматики в нотации БНФ:

### Грамматика в компактной форме (используется в реализации):

Где:
- `<S> `          - начальный символ (функция)
- `<P> `          - список параметров
- `<T>`          - тип данных
- `<E>`           - арифметическое выражение
- ` <O> `          - операнд
- `<Знак_сложения>` - + | -
- `<Знак_умножения>` - * | /

## Классификация грамматики по Хомскому

Данная грамматика относится к **контекстно-свободной грамматике (КС-грамматике, тип 2)** по классификации Хомского.

### Обоснование:

1. Все правила имеют вид `A → α`, где `A` — нетерминал, `α` — строка из терминалов и нетерминалов
2. Левая часть каждого правила состоит ровно из одного нетерминала
3. Правила не зависят от контекста (окружения)
4. Грамматика описывает вложенные структуры (например, выражения со скобками)

### Примеры правил, доказывающих КС-грамматику:

| Правило | Обоснование |
|---------|-------------|
| ` <S> `   → fun id |( <P> ) : <T> { return <E> } ;
| ` <P>  `  → <P_хвост> | ε
| ` <P_хвост>` → id : <T> | id : <T> , <P_хвост>
| ` <T>`    → |Int
| ` <E> `   → <O> | <E> <Знак_сложения> <O>
| ` <O> `   → id | num | ( <E> ) | <O> <Знак_умножения> <O>
|  `<Знак_сложения>` → + | -
|  `<Знак_умножения>` → * | /

Грамматика **не является регулярной (тип 3)** из-за наличия рекурсивных правил и вложенных конструкций. В регулярных грамматиках все правила должны иметь вид `A → aB` или `A → a`, что не выполняется для правил вида `E → ( E )` и `E → E + E`.
### Грамматика в компактной форме (EBNF)
`S ` → fun id ( P ) : T { return E } ;
`P`  → id : T { , id : T }
`T ` → Int
`E ` → O { (+|-) O }
`O ` → id | num | ( E ) | O { (*|/) O }

## Классификация грамматики по Хомскому

### Обоснование:

|Критерий |	Выполнение | Пояснение |
|---------|------------|----------|
|Левая часть состоит из одного нетерминала|	✓	| Все правила имеют вид A → α |
|Отсутствие контекстной зависимости|	✓	|Правила не зависят от окружения|
|Наличие рекурсивных правил|	✓|	E → E + O, O → O * O|
|Вложенные структуры|	✓	|E → ( E ) — скобочные выражения|

### Примеры правил, доказывающих КС-грамматику:

|Правило |	Обоснование |
|---------|------------|
|`E → E + O`	|Леворекурсивное правило, допустимое для КС-грамматик|
|`E → ( E )`	|Вложенная структура, невозможная в регулярных грамматиках|
|`P → id : T , P`|Рекурсивное правило для списка параметров|

## Диагностика и нейтрализация ошибок методом Айронса
### Основные принципы метода
Метод Айронса (Irons, 1968) предназначен для обработки синтаксических ошибок при нисходящем разборе без возврата (backtracking).

Ключевая идея: При обнаружении синтаксической ошибки анализатор НЕ ОСТАНАВЛИВАЕТСЯ, а пропускает «проблемные» символы до тех пор, пока не сможет продолжить разбор с корректного места. Это позволяет обнаружить ВСЕ ошибки в программе за один проход.

## Пример работы метода Айронса

`fun calc(a: Int, b Int, c: Int): Int { return a + (b * c) };`

##Проблема:##  После `b` ожидается `:`, но найдено `Int`.

|Шаг|	Текущий токен	 | Ожидалось | Действие |
|---------|------------|----------|----------|
|1|	`fun	`|`fun`|	✓ Совпало, продолжаем|
|2|	`calc`|	`id`|	✓ Совпало, продолжаем|
|3|	`(`|	`(	`|✓ Совпало, продолжаем|
|4|	`a`|	`id`|	✓ Совпало, продолжаем|
|5|	`:`|	`:`|	✓ Совпало, продолжаем|
|6|	`Int`|	`Int`|	✓ Совпало, продолжаем|
|7|	`,`|	`,` или `)`|	✓ Совпало, продолжаем|
|8|	`b`|	`id`|✓ Совпало, продолжаем|
|9|	`Int`|	`:`| 	❌ ОШИБКА!|

### ДИАГНОСТИКА:

Неверный фрагмент: "Int"

Позиция: строка 1, столбец 16

Описание: Ожидалось ":", найдено "Int"

### НЕЙТРАЛИЗАЦИЯ: 

Пропускаем токен "Int"

Следующий токен: ","

"," входит в синхронизирующие токены для P

Продолжаем разбор с ","

|Шаг|	Текущий токен	 | Ожидалось | Действие |
|---------|------------|----------|----------|
|10|	`,	`|`,` или `)`|	✓ Совпало, продолжаем|
|11|	`c`|	`id`|	✓ Совпало, продолжаем|
|12|	`:`|	`:	`|✓ Совпало, продолжаем|
|13|	`Int`|	`Int`|	✓ Совпало, продолжаем|
|14|	`)`|	`)`|	✓ Совпало, продолжаем|
|15|	`:`|	`:`|	✓ Совпало, продолжаем|
|16|	`Int`|	`Int`|	✓ Совпало, продолжаем|
|17|	`{`|	`{`|✓ Совпало, продолжаем|
|18|	`return`|	`return`| 	❌ ОШИБКА!|

###Результат: Ошибка обнаружена, анализ продолжен до конца строки.
